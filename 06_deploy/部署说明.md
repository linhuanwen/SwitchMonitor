# 道岔监测系统 (SwitchMonitor) 部署说明

## 部署环境

- **目标平台**: 研华 610H 工控机, Windows XP SP3
- **运行框架**: .NET Framework 4.0
## xcopy 部署步骤

1. 将 `build_out/` 目录完整拷贝到工控机（如 `E:\道岔监测\`）
2. **导入 IE8 注册表修复**：双击 `ie8_fix.reg`，确认导入（确保 WebBrowser 控件以 IE8 标准模式渲染）
3. 修改 `config.json` 中的路径配置：
   - `dataSourceDir`: CSV 数据源目录（存放 SwitchCurve(N).csv）
   - `parsedDataDir`: 已解析 JSON 数据目录
4. 确保 `parsed_data/` 目录存在（含 `index.json` 和 8 个道岔组子目录）
5. 双击 `SwitchMonitor.exe` 启动

## 部署文件清单

```
SwitchMonitor.exe              # 主程序 (WebBrowser + Highcharts)
SwitchMonitor.Data.dll         # 数据层 (CSV解析、JSON存储)
SwitchMonitor.pdb              # 调试符号 (可选)
SwitchMonitor.Data.pdb         # 调试符号 (可选)

parsed_data/                   # 已解析数据目录

  1-1/  1-X/  2-1/  2-X/      # 8个道岔组日数据
  3-1/  3-X/  4-1/  4-X/
```

## 与原有软件的共存

- 不修改、不拦截、不依赖原有 CSM 软件的任何内部机制
- 仅读取 CSM 软件导出的 CSV 文件（只读模式）
- 如果 CSM 软件未运行，程序仍可正常浏览历史数据
- 程序异常挂掉不会影响 CSM 软件的数据采集

## XP 兼容性说明

- 程序使用 WebBrowser 控件内嵌 Highcharts 图表
- 通过 `FEATURE_BROWSER_EMULATION = 8888` 强制 IE8 标准模式
- Highcharts 2.2.1 在 IE8 上使用 VML 降级渲染（不支持 SVG）
- 如遇渲染问题或性能不足，可回退至 `08_归档_旧GDI+版\`（纯 GDI+ 自绘方案）
