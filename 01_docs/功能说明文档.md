## 1. 系统概述

### 1.1 项目背景

道岔监测系统 (SwitchMonitor) 是一套**铁路信号集中监测（CSM）道岔动作曲线分析与诊断工具**。现有 CSM 站机软件虽是成熟的商业软件，但存在以下局限性：

- **无法自定义异常诊断规则**：内置报警规则固定，无法根据实际运维经验调整阈值或增加新规则
- **缺少自动诊断能力**：仅提供曲线查看，不对每次动作自动判断是否异常
- **无法进行深入分析**：不支持多曲线叠加对比、转换时间趋势分析、道岔间横向对比
- **缺少参考曲线管理**：无法为每个道岔设定标准参考曲线并标注设定时间

本系统从零构建，与既有 CSM 软件**零侵入并存**运行，在保留原有曲线展示功能的基础上，增加了可配置的异常诊断引擎。

### 1.2 核心功能一览

| 功能模块 | 说明 |
|----------|------|
| 数据采集 | 自动扫描 CSM 软件生成的 `.dat` 二进制文件，解析道岔动作曲线和开关量事件 |
| 曲线展示 | 三相电流/电压/功率曲线叠加显示，支持缩放、拖拽、局部放大 |
| 参考曲线管理 | 为每台道岔设定标准参考曲线，支持历史动作与参考曲线叠加对比 |
| 异常诊断引擎 | 自动分析每次道岔动作的功率曲线，输出 正常/预警/报警/故障 四级结论 |
| 报警展示 | 侧边栏时间列表着色、诊断结论条、日期角标，异常一目了然 |
| 配置管理 | 道岔映射、诊断规则阈值通过 JSON 配置文件管理，现场可编辑无需重编译 |
| 数据导出 | 支持曲线导出为 PNG 图片、原始数据导出为 CSV 文件 |

### 1.3 关键设计原则

- **零侵入**：不修改、不拦截、不依赖原有 CSM 软件的任何内部机制
- **xcopy 部署**：整个程序文件夹拷贝到工控机即可运行，无需安装程序
- **可替换规则**：诊断规则与主程序解耦，更新规则只需替换 JSON 文件或诊断 DLL
- **配置驱动**：道岔名称、编号映射、文件路径等均通过配置文件管理

## 3. 数据采集与解析

### 3.1 数据采集流程

系统通过**文件系统扫描**方式获取数据，不依赖任何网络通信或进程间通信：

```
[后台定时器每 30 秒扫描]
    │
    ├── 扫描数据源目录
    │   ├── 发现新的 SwitchCurve(*).dat → 排入解析队列
    │   └── 发现新的 Digit(*).dat → 排入解析队列
    │
    ├── 对每个新文件调用解析器
    │   ├── SwitchCurveParser.Parse(byte[]) → SwitchActionData 列表
    │   └── DigitParser.Parse(byte[]) → StatusEvent 列表
    │
    ├── 数据入库 (SQLite)
    │   ├── INSERT SwitchActions + CurveSamples
    │   └── INSERT StatusEvents
    │
    └── 触发诊断
        └── 对每个新 SwitchAction → DiagnosisEngine.Diagnose() → 展示结果
```

**关键特性**：
- **增量处理**：已处理过的文件不重复解析，通过 `ProcessedFileRepository` 记录
- **容错机制**：文件损坏或格式不匹配时记录错误日志，不中断整体流程
- **只读访问**：仅读取 CSM 软件写入的文件，不影响原有数据采集

### 3.2 支持的数据类型

#### 3.2.1 CSM2010 道岔动作曲线 (SwitchCurve*.dat)

- **来源**：CSM 站机软件实时采集的道岔动作电气参数
- **内容**：每次道岔动作的三相电流/电压/功率随时间变化的波形
- **采样率**：25 Hz（每 0.04 秒一个采样点）
- **数据量**：三水北站 8 组道岔，16 个文件（每组道岔 2 个文件）

#### 3.2.2 开关量事件 (Digit*.dat)

- **来源**：CSM 站机软件采集的继电器/开关量状态变化
- **内容**：每个采集点的时间戳、点号（point_id）和状态字节（state_byte）
- **数据量**：84 个文件，13,667 条事件（2026-06-26 至 2026-06-29）

#### 3.2.3 模拟量采样 (DCBSDYAnalog*.dat)

- **来源**：CSM 站机软件采集的模拟量（疑似表示电压）
- **内容**：每个采样点的时间戳、通道号、测量类型和浮点数值
- **数据量**：84 个文件，492,672 条采样
- **状态**：V1 暂不纳入主程序展示，V2 计划支持

### 3.3 数据存储 (SQLite)

系统使用 SQLite 单文件数据库存储所有解析后的数据：

| 表名 | 内容 | 说明 |
|------|------|------|
| `SwitchActions` | 道岔动作事件 | 车站、道岔编号、起止时间、方向、采样数 |
| `CurveSamples` | 曲线采样数据 | 每个采样点的时间戳、相别、电流、电压、功率 |
| `StatusEvents` | 开关量状态事件 | 时间戳、点号、状态码、关联道岔 |
| `ReferenceCurves` | 参考曲线 | 道岔ID、来源动作、设定时间、备注 |
| `DiagnosisLog` | 诊断记录 | 动作ID、规则名、级别、结论描述、异常值 |

## 5. 参考曲线管理

### 5.1 功能说明

参考曲线是某道岔在确认正常状态下记录的标准动作曲线，作为后续动作对比的基准。

### 5.2 操作流程

1. 在历史动作列表中选择一次确认正常的动作
2. 点击 **"设为参考曲线"** 按钮
3. 系统记录设定时间和道岔标识
4. 后续查看该道岔动作时，参考曲线以**虚线或半透明**叠加显示

### 5.3 管理功能

- 每个道岔可设定多条参考曲线（如按季度更新）
- 只有一条标记为"当前使用中"（IsActive=1）
- 参考曲线列表显示设定时间和备注信息
- 支持删除或替换旧的参考曲线

## 6. 异常诊断引擎

诊断引擎是系统的核心增值模块，独立于主程序，通过 `IDiagnosisEngine` 接口与主程序通信。

### 6.1 诊断流程

```
历史功率曲线 (23,999条)
       │
       ▼
  ┌─────────────┐     ┌──────────────────┐
  │  基线构建    │────▶│ baselines.json   │  每台道岔的正常标准
  └─────────────┘     └──────────────────┘
                             │
  新动作发生                  │
       │                     │
       ▼                     ▼
  ┌─────────────┐     ┌──────────────────┐
  │  特征提取    │     │ thresholds.json  │  可配置的判定阈值
  └─────────────┘     └──────────────────┘
       │                     │
       └──────────┬──────────┘
                  ▼
         ┌───────────────┐
         │   规则引擎     │  R0-R8 规则评估
         └───────────────┘
                  │
                  ▼
         ┌───────────────┐
         │   诊断结论     │  正常/预警/报警/故障
         └───────────────┘
```

### 6.2 功率曲线特征提取

对每次道岔动作的功率曲线，提取 **12 维特征值**：

| 特征 | 说明 |
|------|------|
| `SampleCount` | 原始采样点数 |
| `IsFullWindow` | 是否打满录制窗口（≥780 点 ≈31.2s） |
| `IsValid` | 曲线是否有效（n > 0 且峰值 > 0.01kW） |
| `ActiveEnd` | 有效动作终点下标（去尾部零填充） |
| `DurationSec` | 动作时长（秒） |
| `SpikePeak` | 启动尖峰最大值 |
| `SpikeIndex` | 启动尖峰所在下标 |
| `UnlockMean` | 解锁段均值 |
| `ConvMean` | 转换段均值 |
| `ConvMax` | 转换段最大值 |
| `StepRatio` | 台阶比（转换段后 1/3 均值 / 前 1/3 均值） |
| `TailMean` | 缓放段均值 |

**五阶段分割**（对应道岔动作的物理过程）：

```
启动尖峰 ──▶ 解锁段 ──▶ 转换段（主体）──▶ 锁闭凹口 ──▶ 缓放尾段
 0.3s         0.5s        5-10s           0.3s         0.9s
```

### 6.3 统计基线

对每台道岔，从全部历史曲线中自动计算正常基线：

1. 筛选正常样本（排除超时、无效、过短曲线）
2. 时长在总体中位数 ±15% 范围内
3. 对剩余样本的每项特征取**中位数**，得到 5 项基线值

| 基线值 | 说明 | 1-1 (J型) 参考值 |
|--------|------|------------------|
| `RefDurationSec` | 参考动作时长 | 11.72s |
| `RefSpikePeak` | 参考启动峰值 | 3.235 kW |
| `RefUnlockMean` | 参考解锁段均值 | 0.307 kW |
| `RefConvMean` | 参考转换段均值 | 0.301 kW |
| `RefTailMean` | 参考缓放段均值 | 0.214 kW |

### 6.4 诊断规则表 (R0-R8)

判定顺序：R0 → R1 → R2 命中即终止；否则 R3-R8 全部评估可多命中。

| ID | 规则名 | 判据 | 级别 | 对应病害 |
|----|--------|------|------|----------|
| R0 | 采集异常 | 曲线无效（`!isValid`） | **报警** | 采集/解析故障 |
| R1 | 动作超时/未完成 | 打满窗口 或 时长 > 参考 + 3.0s | **故障** | 卡阻、空转、异物 |
| R2 | 动作夭折 | 时长 < 参考 × 0.6 | **报警** | 未解锁、保护切断 |
| R3 | 动作时长偏差 | \|时长 − 参考\| > 0.5s | **预警** | 阻力变化早期征兆 |
| R4 | 启动峰值偏高 | 峰值 > 参考 × 1.3 | **预警** | 启动回路/机械卡滞 |
| R5 | 转换段功率偏高 | 转换段均值 > 参考 × 1.3 | **预警** | 滑床板缺油、异物 |
| R6 | 转换段台阶突变 | 台阶比 > 1.5 或 < 0.67 | **报警** | 中途受阻/空转 |
| R7 | 解锁段偏高 | 解锁段均值 > 参考 × 1.3 | **预警** | 密贴过紧、卡缺口 |
| R8 | 缓放段异常 | 缓放段偏离参考 ±30% | **预警** | 锁闭/开闭器异常 |

**事件综合级别** = 命中规则的最高级别；无命中 = 正常。

**演习数据**（三水北站 8 台道岔，2025-12 ~ 2026-06，23,999 事件）：
- 触发 375 条异常（1.56%），≈每台每天 0.2 条报警
- R1（超时）最多：198 次；R2（夭折）：24 次；R5（转换段偏高）：6 次

### 6.5 诊断存储

| 文件 | 内容 | 说明 |
|------|------|------|
| `Rules/thresholds.json` | 9 条规则的启用/级别/阈值参数 | 手工可编辑 |
| `Rules/baselines.json` | 每台道岔的 5 项基线值 + 元数据 | 工具自动生成 |
| `{date}.diag.json` | 该日每个事件的诊断结论 | 与曲线数据并列 |
| `alarms_index.json` | switchId → date → 级别计数 | UI 角标用 |
| `diag.log` | 诊断运行日志 | 含规则评估明细 |

### 6.6 诊断命令行工具 (DiagTool)

控制台程序，支持以下子命令：

| 命令 | 功能 |
|------|------|
| `selftest` | 使用金标准夹具验证特征提取和规则判定 |
| `baseline` | 从全量历史数据构建基线，输出 baselines.json |
| `dryrun` | 对全量数据运行诊断，统计规则触发数 |
| `rerun` | 重新诊断全部数据（阈值调参后使用） |

## 8. 配置管理

### 8.1 道岔映射配置

`05_production_data/Config/switch_mapping.json` — 定义文件名与道岔编号的对应关系：

```json
{
  "version": "1.0",
  "stationId": "SSB",
  "stationName": "三水北",
  "fileMapping": {
    "SwitchCurve(0)": {
```

## 10. 数据格式说明

### 10.1 CSM2010 道岔曲线格式

- **魔数**：`CSM2010\x00`
- **结构**：文件头 + 24 字节索引记录 + 多个数据块
- **数据块**：4B timestamp (LE uint32) + 4B flags + 4B sample_count + 采样值数组
- **flags 含义**：`16777216`=A相, `33554432`=B相, `50332416`=C相, `0`=功率
- **采样率**：25 Hz，每相约 150-790 个采样点
- **采样值类型**：4 字节 float32 LE 或 2 字节 int16 LE

### 10.2 Digit 开关量格式

- **文件头**：含 `ts_start` (offset +12) 和 `ts_end` (offset +16)
- **记录结构**（变长）：
  - 4 字节 timestamp (LE uint32)
  - 2 字节 record_type (BE uint16) — 本记录包含的点数
  - N×2 字节点数据 (BE uint16) — 高字节=state_byte，低字节=point_id
  - 1 字节校验/填充
- **常见状态码**：`0x2f`（吸起）、`0x00`（落下）

### 10.3 DCBSDYAnalog 模拟量格式

- **记录结构**（约 16 字节）：
  - +0：4 字节 timestamp (LE uint32)
  - +4：2 字节 channel（通常为 0）
  - +6：2 字节 measurement_type（1-32）
  - +12：4 字节 float32 value
- **数值范围**：0.00 - 227.68 V（疑似直流电压）

### 10.4 数据库 Schema

详见 [PRD.md](PRD.md) 第 153-224 行，包含 5 张核心表：

- `SwitchActions` — 道岔动作事件
- `CurveSamples` — 曲线采样数据
- `StatusEvents` — 开关量状态事件
- `ReferenceCurves` — 参考曲线
- `DiagnosisLog` — 诊断记录

## 11. 部署说明

### 11.1 部署环境

- **目标平台**：研华 610H 工控机
- **操作系统**：Windows XP SP3 32-bit
- **运行框架**：.NET Framework 4.0
- **目标车站**：三水北 (SSB)

### 11.2 部署步骤

1. 将 `build_out/` 目录完整拷贝到工控机（如 `E:\道岔监测\`）
2. 双击 `ie8_fix.reg`，确认导入注册表（确保 WebBrowser 控件以 IE8 标准模式渲染）
3. 修改 `config.json` 中的路径配置：
   - `dataSourceDir`：CSV 数据源目录
   - `parsedDataDir`：已解析 JSON 数据目录
4. 确保 `parsed_data/` 目录存在（含 `index.json` 和道岔组子目录）
5. 双击 `SwitchMonitor.exe` 启动

### 11.3 部署文件清单

```
SwitchMonitor.exe              # 主程序
SwitchMonitor.Data.dll         # 数据层
SwitchMonitor.Common.dll       # 公共模型
SwitchMonitor.Diagnosis.dll    # 诊断接口
DiagnosisEngine.dll            # 诊断引擎
System.Data.SQLite.dll         # SQLite 驱动
Newtonsoft.Json.dll            # JSON 处理
config.json                    # 运行时配置
ie8_fix.reg                    # IE8 注册表修复
Rules/                         # 规则配置目录
parsed_data/                   # 已解析数据目录
```

### 11.4 与原有软件的共存

- 不修改原有软件的任何文件、配置或注册表
- 仅读取 CSM 软件写入的 `.dat` 文件（只读模式）
- CSM 软件未运行时，程序仍可正常浏览历史数据
- 程序异常不会影响 CSM 软件的数据采集
