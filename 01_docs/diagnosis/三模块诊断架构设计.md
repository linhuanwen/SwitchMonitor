> **版本**: V1.0
> **日期**: 2026-07-13
> **设计者**: 林焕文
> **状态**: 架构设计完成，待实现

---

## 1. 设计动机

当前诊断系统（R0-R8 + T1/P1）在**显性问题**（持续超阈值、超时、夭折）的检测上已较为完善。但存在两类根本性不足：

1. **隐性问题盲区**：转换过程中出现瞬时功率脉冲（阻力点），转辙机凭借余量克服后恢复正常——整段统计量被"平均掉"，固定阈值无法触发。
2. **知识来源单一**：当前所有规则来自领域专家经验（先验知识），未体系化地引入学术文献、技术标准中的研究成果。

## 3. Module 1 — 参考曲线标定

### 3.1 定位

最轻量的模块。纯粹的数据记录 + 可视化叠加，**不涉及诊断逻辑**。为 M2 的 P1（逐点残差对比）和 M3 中任何需要"正常模板"的方法提供数据基础。

### 3.2 数据流

```
运维人员选中一条"确认正常"的曲线
         │
         ▼
  点击 UI "设为参考曲线"
         │
         ▼
  保存: Rules/reference_curves/{switchId}.json
         │
         ▼
  后续查看该道岔任何动作时 → 图表叠加显示参考曲线(虚线/半透明)
  P1逐点对比 → 使用该参考曲线计算残差面积比
```

### 3.3 存储格式

```json
{
  "switchId": "1-1",
  "sourceTimestamp": 1770922311,
  "sourceDate": "2026-02-13",
  "setBy": "运维人员（可选）",
  "setReason": "2026Q1季度标定 | 检修后标定 | 手动标定",
  "sampleInterval": 0.04,
  "spikeIndex": 6,
  "values": [0.0, 0.0, 0.14, 0.24, 1.85, 3.10, 3.39, "...N个3位小数"],
  "createdAt": "2026-02-15T10:30:00"
}
```

### 3.4 与 baselines.json 的区别

| | baselines.json (D2) | reference_curves/ (M1) |
|---|---|---|
| 内容 | 5个标量中位数值 | 完整的逐点采样波形 |
| 来源 | 30+条正常样本聚合 | 单条标定曲线 |
| 用途 | R3-R13的阈值比较基线 | P1逐点残差 + 图表叠加显示 |
| 更新 | 季度重算 | 手动标定（任意时刻） |

## 4. Module 2 — 先验知识规则引擎（生产区）

### 4.1 定位

诊断系统的**生产规则引擎**。所有在此模块中的规则均已经过验证，直接参与每次动作的诊断判定。规则来源有两类：

- **原始领域专家规则**：来自铁路信号运维经验，直接编码为 R 系列规则
- **M3 晋升规则**：来自文献知识库的候选规则，经 dryrun 验证 + 人工审查通过后晋升

### 4.2 规则分类

| 层级 | 规则 | 类别 | 说明 |
|------|------|------|------|
| L1 完整性 | R0-R2 | integrity | 不依赖基线，检测数据完整性和动作完成性 |
| L2 分阶段阈值 | R3-R8 | threshold | 各段统计量 vs 基线，检测持续偏离 |
| L2+ 隐性检测 | R9-R13 | morphology | 段内局部特征，检测被平均值掩盖的隐性问题 |
| L3 趋势形态 | T1, P1 | trend | 多事件趋势 + 逐点对比 |

### 4.3 完整规则矩阵

| ID | 规则名 | 判据 | 级别 | 检测类型 | 状态 |
|----|--------|------|------|----------|------|
| R0 | 采集异常 | `!isValid` | 报警 | 显性 | ✅ 已实现 |
| R1 | 动作超时/未完成 | `isFullWindow ∨ dur>refDur+3.0s` | 故障 | 显性 | ✅ 已实现 |
| R2 | 动作夭折 | `dur<refDur×0.6` | 报警 | 显性 | ✅ 已实现 |
| R3 | 动作时长偏差 | `|dur−refDur|>0.5s` | 预警 | 显性 | ✅ 已实现 |
| R4 | 启动峰值偏高 | `spikePeak>ref×1.3` | 预警 | 显性 | ✅ 已实现 |
| R5 | 转换段功率偏高 | `convMean>ref×1.3` | 预警 | 显性 | ✅ 已实现 |
| R6 | 转换段台阶突变 | `stepRatio>1.5∨<0.67` | 报警 | 显性 | ✅ 已实现 |
| R7 | 解锁段偏高 | `unlockMean>ref×1.3` | 预警 | 显性 | ✅ 已实现 |
| R8 | 缓放段异常 | `tailMean偏离ref±30%` | 预警 | 显性 | ✅ 已实现 |
| **R9** | **转换段瞬态阻力** | `ConvTransientScore>1.5` | 预警 | **隐性** | 🔴 待实现 |
| **R10** | **转换段稳定性异常** | `ConvRoughness>ref×2.0` | 预警 | **隐性** | 🔴 待实现 |
| **R11** | **解锁/转换比例失调** | `|ratio−ref|>0.2` | 预警 | **隐性** | 🔴 待实现 |
| **R12** | **尖峰半高宽异常** | `SpikeHalfWidth偏离±40%` | 预警 | **隐性** | 🔴 待实现 |
| **R13** | **尖峰回落不足** | `SpikeDropRatio<0.7` | 预警 | **隐性** | 🔴 待实现 |
| T1 | 渐变劣化趋势 | 7天滑动中位+单调不减 | 预警 | 隐性 | 🔴 待实现 |
| P1 | 逐点形态偏离 | `areaDiffRatio>0.25` | 预警 | 隐性 | 🔴 待实现 |

### 4.4 判定顺序

```
R0 命中? ──Yes──▶ 终止, 输出"报警"
  │No
  ▼
R1 命中? ──Yes──▶ 终止, 输出"故障"
  │No
  ▼
R2 命中? ──Yes──▶ 终止, 输出"报警"
  │No
  ▼
R3-R13 + T1 + P1 全部并行评估（可多命）
  │
  ▼
综合级别 = max(命中规则的级别)
```

### 4.5 关键设计决策

**加性 vs 乘性阈值**：
- 时长特征选**加性**（±0.5s）："秒"有绝对物理含义
- 功率特征选**乘性**（×1.3）：功率与负载成正比，同比例 = 同程度异常
- 比例特征选**绝对差值**（±0.2）：比值无量纲，相对偏差不适用

**L1 序列终止逻辑**：R0-R2 命中后不继续评估 L2。原理：超时/夭折曲线的后续特征必然异常，叠加报警只会产生冗余噪音。

## 5. Module 3 — 文献知识库（孵化区）

### 5.1 定位

诊断系统的**规则孵化引擎**。学术论文、技术标准、行业规范中的诊断知识在此被：
1. 结构化为**知识条目**（Knowledge Entry）—— 记录论文方法、关键发现、可迁移性评估
2. 转化为**候选规则**（Candidate Rule）—— 适配到我们的特征框架，默认禁用
3. 经 dryrun 验证 + 人工审查后 → **晋升至 M2**（生产规则）

## 6. 新增特征定义（支撑 R9-R13）

为支撑隐性问题的检测，FeatureExtractor 需新增以下特征：

### 6.1 ConvTransientScore（R9）

```
输入: convSegment[m], windowSize=20
算法:
  scores = []
  for i = 0 .. m-windowSize:
    localMean = mean(convSegment[i : i+windowSize])
    scores.append(localMean / refConvMean)
  ConvTransientScore = max(scores)
  ConvTransientIndex = argmax(scores) × 0.04  # 异常位置(秒)
```

**物理含义**：转换段内任何 0.8s 窗口的平均功率相对于整段基线的最大偏离倍数。>1.5 即存在局部阻力点。

### 6.2 ConvRoughness（R10）

```
输入: convSegment[m], movingMedianWindow=11
算法:
  trend = moving_median(convSegment, 11)     # 去趋势
  residuals = convSegment[i] - trend[i]
  ConvRoughness = std(residuals) / convMean  # 归一化粗糙度
```

**物理含义**：转换段功率的"抖动"程度。液压系统正常情况下应保持平稳输出；粗糙度增大说明油压不稳。

### 6.3 UnlockToConvRatio（R11）

```
UnlockToConvRatio = unlockMean / convMean
refUnlockToConvRatio = refUnlockMean / refConvMean  （基线中新增）
```

**物理含义**：解锁所用功率与转换所用功率的比例。正常值约 1.0（解锁略高）。比例失调提示某一环节负载异常。

### 6.4 SpikeHalfWidth（R12）

```
riseHalf  = 第一个 v[i] > spikePeak×0.5 的下标 (i ≤ spikeIndex)
fallHalf  = 第一个 v[i] < spikePeak×0.5 的下标 (i ≥ spikeIndex)
SpikeHalfWidth = (fallHalf - riseHalf) × 0.04  # 秒
```

**物理含义**：启动尖峰的持续时间（半高全宽），反映油泵建压和溢流阀响应的综合速度。

### 6.5 SpikeDropRatio（R13）

```
postSpikeMin = min(v[spikeIndex : spikeIndex+10])
SpikeDropRatio = (spikePeak - postSpikeMin) / spikePeak
```

**物理含义**：尖峰后功率回落的幅度占比。正常应回落约 90%（到解锁段水平）。<0.7 说明回落不足，溢流阀或管路异常。

## 8. 实施路线

```
Phase 1 — 基础设施（1-2天）
├── FeatureExtractor 扩展: 新增 ConvTransientScore/ConvRoughness/
│   UnlockToConvRatio/SpikeHalfWidth/SpikeDropRatio 五个特征
├── BaselineBuilder 扩展: baselines.json 新增 refUnlockToConvRatio
│   refRoughness/refSpikeHalfWidth/refSpikeDropRatio 四项基线
└── DiagTool.exe 支持 --include-literature 标志加载M3候选规则

Phase 2 — 核心规则落地（1-2天）
├── R9  转换段瞬态阻力 — 滑动窗口扫描（最优先，直接命中隐性瞬时阻力）
├── R11 解锁/转换比例失调 — 一个除法（最简单，性价比最高）
├── R12 尖峰半高宽异常 — 两个二分搜索
```

## 9. 附录

### A. 显性问题 vs 隐性问题 形式化定义

| | 显性问题 (Explicit) | 隐性问题 (Implicit) |
|---|---|---|
| **定义** | 异常信号在整段统计量层面表现为明显、持续的偏离 | 异常信号在局部时间窗口内显著存在，但被整段均值"平均掉" |
| **机制** | 问题的严重程度已大到转辙机自身无法掩盖 | 转辙机凭借远大于正常负载的转换力强行克服了阻力点 |
| **曲线表现** | 整段均值超阈值、动作超时、夭折 | 段内瞬态脉冲、局部抖动、段间比例失调 |
| **当前覆盖** | R0-R8 ✅ | R9-R13 🔴 (本文设计) |
| **典型场景** | 卡阻空转、滑床板严重缺油、保护切断 | 轨缝异物被挤开、滑床板局部干涩、液压气泡 |
| **检测方法** | 固定阈值 vs 基线中位数 | 滑动窗口、残差分析、段间比例、尖峰形态 |

### B. 为什么 M3 规则需要 dryrun 而非直接启用

1. **设备差异**：论文多基于 S700K（电动），我们的设备是 ZYJ7（电液）。功率曲线形态存在系统性差异。
2. **采样率差异**：论文多用 50Hz 或更高采样率，我们是 25Hz。时间参数需折算。
3. **数据分布差异**：论文多来自实验室故障模拟平台，我们的数据来自现场实际运营。故障模式、噪声水平均不同。
4. **诊断范式差异**：论文多为多分类（需要各类故障样本），我们为 one-class 异常检测（仅需正常样本建基线）。论文的"故障A vs 故障B vs 正常"的判别边界不能直接移植为"正常 vs 异常"的阈值。
