# 标准曲线融合模块设计 (StandardCurveBuilder)

> **版本**: V3.0
> **日期**: 2026-07-14
> **设计者**: 林焕文
> **状态**: 设计完成，待实现

---

## 1. 设计动机

当前系统已有两项数据资产：

| 资产 | 内容 | 来源 | 粒度 |
|---|---|---|---|
| **基线** (`baselines.json`) | 6 个标量中位数值 | `BaselineBuilder` 从数千条正常曲线自动聚合 | 标量 |
| **参考曲线** (`reference_curves/{switchId}.json`) | 完整逐点功率波形 | 电务人员手动指定单条"确认正常"曲线 | 波形 |

二者各有局限：

- **基线**（6 个标量）无法重建完整波形——无法回答"锁闭凹口的精确形状是什么""转换段内的特征性波动长什么样"
- **参考曲线**（单条标杆）形态完整，但缺乏统计稳健性——那条曲线可能恰好液压油温偏低导致整体偏高 5%，电务人员肉眼无法察觉

**融合思路**：以人工参考曲线为**形态模板**，以统计基线为**水平锚定**，生成一条兼具形态保真与统计稳健的"标准曲线"。

---

## 2. 设计定论：基线不能扔

经过对基线角色、近邻漂移、故障案例的充分讨论，最终结论：

**基线是灯塔（独立统计锚点），参考曲线是形态模板，两者解决不同问题，缺一不可。**

| 组件 | 提供什么 | 防止什么 |
|---|---|---|
| **基线** (baselines.json) | 3000 条正常曲线的统计中心 → "正常应该是什么水平" | 漂移中毒（系统跟着缓慢劣化一起漂走） |
| **参考曲线** (reference_curves/) | 单条确认正常的完整波形 → "正常曲线长什么样" | 无法检测形态异常（锯齿波等非幅值故障） |
| **标准曲线** (standard_curves/) | 形态 = 参考曲线，水平 = 基线校准 → 兼具两者优势 | — |
| **近邻漂移** (Layer 2) | 最近 20 条正常曲线的局部水平 → "今天这个温度的正常水平" | 季节性/日内温度导致的基线系统性偏差 |

**为什么基线不能扔？** 实际故障案例（参考 `曲线巡检2022-6-23.docx`）中，9 类故障有 7 类标注"有趋势过程"——滑床板缺油、密贴过紧、锁舌回缩等都是渐进劣化。如果只用近邻漂移（局部中位数），系统会跟着劣化信号一起漂走，永远发现不了缓慢的异常爬升。基线是唯一不被近期数据污染的独立锚点。

## 3. 核心概念

```
人工参考曲线 (单条，形态可信)  ×  统计基线 (数千条，统计可靠)
         │                              │
         │  提供：波形形态               │  提供：各段"正确答案"
         │  （凹口深度、过渡斜率、       │  （时长应该是多少、
         │   转换段内纹波特征…）          │   各段功率均值应该是多少…）
         │                              │
         └────────── 融合拟合 ───────────┘
                      │
                      ▼
            标准曲线 (StandardCurve)
            · 形态来自人工标杆
            · 水平对齐到统计中心
            · 可直接用于 P1 逐点对比诊断
            · 可叠加近邻漂移做日内温度适应
```

## 4. 整体架构：双层适应

```
┌─────────────────────────────────────────────────────────────────┐
│                    离线阶段（一次性 / 定期）                      │
│                                                                 │
│  电务人员"设为参考曲线"                                          │
│       │                                                         │
│       ▼                                                         │
│  StandardCurveBuilder.Build(referenceCurve, baseline)            │
│       │                                                         │
│       │  参考曲线形态 × 基线统计校准 = 标准曲线                   │
│       ▼                                                         │
│  standard_curves/{switchId}.json  ← 持久化                       │
│                                                                 │
│  Layer 1 — 基线滑动窗口更新（每两周，最近 500 条正常事件）         │
│       │                                                         │
│       └─▶ BaselineBuilder 重算 → 重新融合标准曲线               │
│           解决：季节性漂移（冬→夏液压油粘度变化）                  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    在线阶段（每次事件到达）                       │
│                                                                 │
│  新事件 E 到达                                                   │
│       │                                                         │
│       ├─▶ Step A: 加载标准曲线 S                                 │
│       │                                                         │
│       ├─▶ Step B: Layer 2 — 从最近 20 条正常事件估计逐段 drift   │
│       │      drift_conv = median(近邻.ConvMean) / S.ConvMean     │
│       │      drift_lock  = median(近邻.LockMean)  / S.LockMean   │
│       │      生成当日调整版标准曲线 S'                            │
│       │      解决：日内温变（中午 vs 晚上、晴天 vs 雨天）          │
│       │                                                         │
│       ├─▶ Step C: P1 — E vs S' 逐点对比                         │
│       │      areaDiffRatio > 0.25 → 形态异常报警                 │
│       │      （用 S' 因为逐点对比需温度适应）                      │
│       │                                                         │
│       └─▶ Step D: R4-R8 — E 各段均值 vs baseline 原值            │
│              ConvMean / baseline.RefConvMean > 1.3 → 0x0005      │
│              ConvMean / baseline.RefConvMean > 1.5 → 0x005+      │
│              （用 baseline 原值因为段均值需固定锚点防漂移中毒）     │
└─────────────────────────────────────────────────────────────────┘
```

**关键设计原则**：
- **P1 逐点对比** → 用 S'（drift 调整后的标准曲线），因为逐点对比需温度适应，否则冬天参考曲线会误报夏天的正常曲线
- **R4-R8 段均值** → 用 baseline 原值（固定锚点），因为段均值需要客观基准才能检测渐变劣化趋势
- **双锚点分离** → P1 跟天气走，R4-R8 不跟天气走——这样既不会因天热误报，也不会因劣化漂移漏报

## 5. 算法设计

### 5.1 输入

| 输入 | 类型 | 来源 |
|---|---|---|
| `referenceCurve` | `ReferenceCurve` | `Rules/reference_curves/{switchId}.json` |
| `baseline` | `SwitchBaseline` | `Rules/baselines.json` 中对应 switchId+direction 条目 |
| `fusionWeight` | `double` (0~1) | 配置参数，默认 1.0 |

### 5.2 输出

`StandardCurve` POCO，写至 `Rules/standard_curves/{switchId}.json`。

### 5.3 算法流程

```
Step 1 ──▶ FeatureExtractor.Extract(referenceCurve.Values)
           获取参考曲线的五阶段分割：spikeIndex, activeEnd, 各段均值

Step 2 ──▶ 计算缩放因子
           α_t      = B.RefDurationSec / R.DurationSec
           α_spike  = B.RefSpikePeak   / R.SpikePeak
           α_unlock = B.RefUnlockMean  / R.UnlockMean
           α_conv   = B.RefConvMean    / R.ConvMean
           α_lock   = B.RefLockMean    / R.LockMean
           α_tail   = B.RefTailMean    / R.TailMean

           每个 α clamp 到 [0.7, 1.3]，再按 fusionWeight 混合：
           α = 1.0 + (α_raw - 1.0) × fusionWeight

Step 3 ──▶ 时间轴线性重采样
           targetLen = round(B.RefDurationSec / 0.04)
           resampled = ResampleLinear(R.Values, targetLen)

Step 4 ──▶ FeatureExtractor.Extract(resampled)
           获取重采样后的段边界（spikeIdx', activeEnd', convStart', …）

Step 5 ──▶ 逐点幅度缩放 + 段边界平滑过渡
           对每个点 i 赋 α(i)，按所在段取对应 α
           段边界 ±3 点内线性混合相邻段 α

Step 6 ──▶ 输出 StandardCurve
```

### 5.4 逐点 α 分配规则

五阶段边界定义（基于重采样后的特征）：

```
段:          ┊ Unlock              ┊ Conv                     ┊ Lock           ┊ Tail
边界:        ┊ [si+2, si+14)       ┊ [si+20, ae-40)          ┊ [ae-40, ae-22) ┊ [ae-22, ae-2)
α:           ┊ α_unlock            ┊ α_conv                   ┊ α_lock         ┊ α_tail
```

过渡区（段间跳跃带，3 点线性混合）：

```
[0, si]           α_spike                          （爬升段统一用尖峰因子）
(si, si+2)        α_spike → α_unlock  线性过渡
[si+2, si+14)     α_unlock                        （解锁段内部）
[si+14, si+20)    α_unlock → α_conv  线性过渡
[si+20, ae-40)    α_conv                          （转换段内部）
ae-40 ± 3          α_conv → α_lock    线性过渡
[ae-40, ae-22)    α_lock                          （锁闭段内部）
ae-22 ± 3          α_lock → α_tail    线性过渡
[ae-22, ae-2)     α_tail                          （缓放段内部）
[ae-2, end)        α_tail                          （尾部保持）
```

其中 `si = spikeIndex'`，`ae = activeEnd'`。

### 5.5 边界条件

| 情况 | 处理 |
|---|---|
| `RefLockMean == 0`（短曲线无锁闭段） | α_lock = 1.0 |
| `RefTailMean == 0`（短曲线无缓放段） | α_tail = 1.0 |
| 参考曲线某段均值为 0 | α 对应的段 = 1.0（除零保护） |
| `fusionWeight = 0` | 所有 α = 1.0，输出=原参考曲线 |
| `fusionWeight = 1` | 完全对齐基线（默认行为） |
| α 超过 [0.7, 1.3] | clamp 到边界 |

### 5.6 重采样算法（ResampleLinear）

```
输入: src (N 个点), targetCount
输出: 长度为 targetCount 的 List<double>

对 k = 0..targetCount-1:
  x = k × (N-1) / (targetCount-1)       // 在连续域 [0, N-1] 中的位置
  left  = floor(x),  right = min(ceil(x), N-1)
  frac  = x - left
  result[k] = src[left] × (1-frac) + src[right] × frac

特殊情况: targetCount=1 时取 src[0]; N=1 时全取 src[0]
```

## 6. 数据结构

### 6.1 StandardCurve POCO

```csharp
public class StandardCurve
{
    /// <summary>道岔标识</summary>
    public string SwitchId;

    /// <summary>动作方向</summary>
    public string Direction;

    /// <summary>采样间隔（秒），通常 0.04</summary>
    public double SampleInterval;

    /// <summary>对齐基准下标（重采样后的 spikeIndex）</summary>
    public int AlignIndex;

    /// <summary>逐点功率值（kW），保留 3 位小数</summary>
    public List<double> Values;

    // ── 融合溯源 ──

    /// <summary>融合权重 0~1</summary>
    public double FusionWeight;

    /// <summary>来源参考曲线的 switchId</summary>
    public string ReferenceSource;

    /// <summary>来源基线的计算时间</summary>
    public string BaselineComputedAt;

    // ── 审计追踪：各段实际应用的缩放因子 ──

    public double AlphaTime;
    public double AlphaSpike;
    public double AlphaUnlock;
    public double AlphaConv;
    public double AlphaLock;
    public double AlphaTail;

    /// <summary>计算时间 yyyy-MM-dd HH:mm:ss</summary>
    public string ComputedAt;
}
```

### 6.3 StandardCurveBuilder

```csharp
public static class StandardCurveBuilder
{
    /// <summary>
    /// 将人工参考曲线与统计基线融合，生成标准曲线。
    /// </summary>
    /// <param name="referenceCurve">人工设定的参考曲线（单条标定波形）</param>
    /// <param name="baseline">统计基线（数千条曲线的中位数）</param>
    /// <param name="fusionWeight">融合强度：0=保持原参考曲线, 1=完全对齐基线（默认）</param>
    /// <param name="clampMin">α 下限</param>
    /// <param name="clampMax">α 上限</param>
    /// <param name="blendHalfWidth">段边界过渡半宽（点数）</param>
    /// <returns>标准曲线；参考曲线无效时返回 null</returns>
    public static StandardCurve Build(
        ReferenceCurve referenceCurve,
        SwitchBaseline baseline,
        double fusionWeight = 1.0,
        double clampMin = 0.7,
        double clampMax = 1.3,
        int blendHalfWidth = 3
    );
}
```

### 6.4 DriftEstimator（Phase 2 — 近邻漂移估计）

```csharp
/// <summary>
/// 各段的漂移估计系数。>1 表示当前水平高于标准曲线，<1 表示低于标准曲线。
/// </summary>
public class SegmentDrift
{
    public double DriftSpike;
    public double DriftUnlock;
    public double DriftConv;
    public double DriftLock;
    public double DriftTail;
    public int NeighborCount;          // 参与计算的近邻数
    public string ComputedAt;          // 计算时间
}

public static class DriftEstimator
{
    /// <summary>
    /// 从最近 N 条正常事件的特征中估计各段当前漂移系数。
    /// drift_seg = median(近邻.segMean) / standardCurve 对应段均值
    /// 近邻不足 N 条时返回全 1.0（无调整）。
    /// drift clamp 到 [0.85, 1.15]，防止单条异常事件污染。
    /// </summary>
    /// <param name="standardCurve">标准曲线（用于提取各段均值基准）</param>
    /// <param name="recentNormalFeatures">最近 N 条正常事件的特征（按时间倒序）</param>
    /// <param name="neighborCount">近邻数量，默认 20（覆盖约 1-2 天）</param>
    public static SegmentDrift Estimate(
        StandardCurve standardCurve,
        List<CurveFeatures> recentNormalFeatures,
        int neighborCount = 20
    );

    /// <summary>
    /// 将各段 drift 应用到标准曲线，生成当日温度调整版。
    /// 内部复用 StandardCurveBuilder 的逐点 α 分配逻辑。
    /// </summary>
    /// <param name="baseCurve">原始标准曲线</param>
    /// <param name="drift">漂移系数</param>
    /// <returns>调整后的标准曲线（Values 已按 drift 逐点缩放）</returns>
    public static StandardCurve ApplyDrift(
        StandardCurve baseCurve,
        SegmentDrift drift
    );
}
```

**DriftEstimator 的使用时机**：每次诊断事件到达时，P1 对比前调用 `Estimate()` + `ApplyDrift()`，生成当日调整版标准曲线 S'。R4-R8 不使用 drift 调整——继续用 baseline 原值做阈值比较。

**与 StandardCurveBuilder 的关系**：`ApplyDrift()` 的逐点 α 分配算法与 `StandardCurveBuilder.Build()` 的 Step 5 一致——按五阶段边界分配 α，段边界 ±3 点线性过渡。区别在于 α 的来源：Build 的 α 来自 baseline 比值，ApplyDrift 的 α 来自近邻漂移比值。

## 7. 关键设计决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 基线是否保留 | ✅ 保留 | 防止漂移中毒；"有趋势过程"故障占 7/9 |
| fusionWeight 默认值 | **1.0**（完全对齐基线） | 基线 3000 样本统计稳健性远超单条参考曲线 |
| P1 模板用 baseline 还是 drift 调整版 | **drift 调整版 S'** | 逐点对比需温度适应，否则冬天参考曲线对夏天曲线误报 |
| R4-R8 锚点用 baseline 还是 drift | **baseline 原值** | 段均值需要固定锚点才能检测渐变劣化趋势 |
| Layer 1（滑动窗口）更新频率 | **每两周，500 条** | 覆盖约 30 天数据，足够感知季节性变化而不频繁抖动 |
| Layer 2（近邻漂移）样本数 | **20 条** | 覆盖约 1-2 天，足以反映当前温度水平 |
| drift clamp 范围 | **[0.85, 1.15]** | 比 baseline 的 [0.7, 1.3] 更紧——日内温变不应超过 ±15% |

## 9. 融合权重设计

`fusionWeight` 控制基线对标准曲线的影响程度：

| fusionWeight | 效果 | 适用场景 |
|---|---|---|
| 0.0 | 标准曲线 = 参考曲线原样（所有 α=1.0） | 纯信任人工选择，不做校准 |
| 0.5 | 各段水平和时长取参考曲线和基线的中间值 | 两者各信一半 |
| **1.0** | **完全对齐基线**（默认） | 统计基线有数千条支撑，远比单样本可靠 |

推荐默认 1.0——基线基于 ~3000 条正常曲线，统计稳健性远高于单条人工标杆。

## 10. 与现有系统的集成

### 10.1 产生时机

```
电务人员通过 UI "设为参考曲线"
        │
        ▼
保存 reference_curves/{switchId}.json
        │
        ▼
【触发】StandardCurveBuilder.Build(referenceCurve, baseline)
        │
        ▼
保存 standard_curves/{switchId}.json
        │
        ▼
图表自动加载标准曲线叠加显示
P1 规则自动切换为使用标准曲线
```

### 10.2 与 P1 规则的关系

P1（逐点形态偏离）目前使用 `reference_curves/` 中的参考曲线做对比模板。标准曲线生成后，P1 的对比流程变为：

```
1. 加载标准曲线 S
2. 调用 DriftEstimator.Estimate(S, 最近 20 条正常特征) → drift
3. 调用 DriftEstimator.ApplyDrift(S, drift) → S'（当日调整版）
4. 用 S' 做 P1 逐点对比（API 完全相同：Values + AlignIndex）
```

优势：
- 各段水平已校准到统计中心 + 日内温度适应，正常波动不会因参考曲线本身偏差而误报
- 形态信息完整保留，能检测锯齿波等非幅值异常
- 冬天设定的参考曲线不会在夏天误报

### 10.3 与 R4-R8 规则的关系

R4-R8（段均值阈值比较）继续使用 baseline 原值作为比较基准：

```
ConvMean / baseline.RefConvMean > 1.3 → 0x0005（转换阻力升高）
ConvMean / baseline.RefConvMean > 1.5 → 0x005+（二次报警，更危险）
LockMean  / baseline.RefLockMean  > 1.3 → 0x0007（锁闭阻力异常）
LockMean  / baseline.RefLockMean  > 1.5 → 0x007+（二次报警）
```

R4-R8 不使用 drift 调整——因为固定锚点是检测渐变劣化的关键。如果阈值也跟着 daily drift 走，系统就能被缓慢爬升的异常"欺骗"。

### 10.4 基线更新后的再融合

当 `baselines.json` 因累积更多数据而重算后（每两周，Layer 1 滑动窗口）：

- 保留原参考曲线不变
- 用新基线重新跑 `StandardCurveBuilder.Build()`
- 覆盖 `standard_curves/{switchId}.json`
- 审计字段自动记录新旧基线版本

## 11. 验证方案

### 11.1 算法正确性

取 1-J 的一条已知正常曲线作为"人工参考曲线"，搭配 `baselines.json` 中 1-J 的基线值运行 Build()：

- [ ] 输出的 `Values` 长度 ≈ `baseline.RefDurationSec / 0.04`
- [ ] 逐点值无 NaN / ±Infinity
- [ ] `fusionWeight=0` 时，输出经 `FeatureExtractor` 再提取，各段均值 ≈ 原始参考曲线的各段均值（误差 < 1%）
- [ ] `fusionWeight=1` 时，输出经 `FeatureExtractor` 再提取，各段均值 ≈ 基线对应 Ref 值（误差 < 3%，因 clamp 和重采样引入微小偏差）
- [ ] 逐点查看：锁闭凹口的形状保留，转换段内波动形态保留
- [ ] clamp 生效：α 不超过 [0.7, 1.3]

### 11.2 模板代表性

对 1-J 生成标准曲线后，随机取 30 条已知正常的 1-J 曲线，逐条与标准曲线计算 P1 的 `areaDiffRatio`：

- [ ] 30 条正常曲线的 `areaDiffRatio` 中位数 < 0.15（标准曲线应能代表正常曲线）
- [ ] 与直接用参考曲线做模板相比，误报率不升高

### 11.3 视觉验证

在同一张 Highcharts 图上叠加显示：
- 原参考曲线（蓝色虚线）
- 标准曲线（红色实线）
- 30 条正常曲线（灰色半透明）

确认标准曲线保持了参考曲线的形态特征，同时各段水平正确偏移到了统计中心。

## 附录 B：参考曲线 vs 统计参考曲线的区别

本项目有两种"参考曲线"，概念不同，避免混淆：

| | 统计参考曲线 | 人工参考曲线 |
|---|---|---|
| 定义 | `ReferenceCurveBuilder.Build()` 的输出——多条曲线按 spikeIndex 对齐后逐点取中位数 | 电务人员手动指定某一条"确认正常"的实际曲线 |
| 来源 | 自动聚合 | 人工选择 |
| 形态 | 模糊（中位数抹平了锐利特征） | 锐利（保留了单条曲线的全部形态细节） |
| 目前使用 | 未启用（`reference_curves/` 为空） | 计划中的 StandardCurveBuilder 输入 |
| 适用场景 | 无人工介入的全自动统计 | 融合基线的形态模板 |

**StandardCurveBuilder 使用的是人工参考曲线**——因为融合的核心价值在于保留形态细节，统计参考曲线的中位数聚合已经丢失了这些细节。
