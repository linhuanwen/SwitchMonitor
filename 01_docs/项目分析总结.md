# 三水北/本地接收数据 项目分析总结

> 本文档汇总了对现场监测软件及相关数据文件的解析成果、关键发现与后续工作建议。

---

## 1. 项目背景

本工作围绕一套铁路信号集中监测系统（CSM）及其产生的多类数据展开，主要涉及：

- **现场监测软件**：位于 `本地接收目录/`，是一套 CSM 站机程序（`CsmthStation.exe` 等），负责采集、显示、存储和回放信号设备状态。
- **数据文件**：位于 `shuju/`，包含多类监测数据：
  - `SwitchCurve(*).dat`：CSM2010 道岔动作曲线
  - `shiqi/`：ASCII 格式的电流、电压、功率、功率因数等
  - `sanshuibei/`：另一组 CSM2010 道岔曲线数据
  - `本地接收目录表示/`、`本地接收目录扳动/`：表示电压模拟量与开关量事件
- **配置文件**：`shuju/Station_SSB/`（三水北）及 `本地接收目录/DataFile/Station_*/` 各站配置。

## 3. 已开发的解析工具

### 3.1 `parse_csm2010.py`

- 读取 `.dat` 文件中的 `CSM2010\x00` 魔数头
- 解析索引记录与数据块
- 输出 CSV：`timestamp, datetime, phase, s0, s1, ...`
- 适用于 `SwitchCurve` 类道岔曲线文件

### 3.2 `parse_shiqi.py`

- 遍历 `shiqi/` 目录树，按日期/设备/事件组织
- 解析 `.elc`（三相电流）、`.vol`（三相电压）、`.pow`（功率）、`.factor`（功率因数×1000）、`.rst`
- 生成：
  - `_summary.csv`：每事件最大值汇总
  - `_combined.csv`：长格式合并数据
  - 每个事件一个独立 CSV

### 3.3 `parse_local_receive.py`

- 解析 `本地接收目录表示/` 下的 `DCBSDYAnalog(*).dat`
- 解析 `本地接收目录扳动/` 下的 `Digit(*).dat`
- 输出对应 CSV 到 `*_parsed/` 目录

## 4. 数据格式说明

### 4.1 CSM2010 道岔曲线（`SwitchCurve*.dat`）

- 文件头：`CSM2010\x00`
- 包含索引记录（24 字节）定位各数据块
- 数据块：4 字节时间戳 + 4 字节标志 + 4 字节采样数 + 采样值（i16 或 f32）
- 每个事件可能包含 1 相或 3 相数据
- 采样率默认 25 Hz

### 4.2 `shiqi/` ASCII 数据

- 每个事件由 5 个同基名文件组成：
  - `.elc`：三相电流（A=/B=/C= 三行）
  - `.vol`：三相电压（A=/B=/C= 三行）
  - `.pow`：有功功率（单行动态数组）
  - `.factor`：功率因数×1000（单行动态数组）
  - `.rst`：未知物理量（非简单无功/视在功率）
- 时间跨度：2021-09-29 至 2021-10-14

### 4.3 `sanshuibei/` 数据

- 16 个 `.dat` 文件，对应 8 组道岔
- 编号规律：
  - 3 相电流文件：0, 4, 8, 12, 16, 20, 24, 28
  - 配对的 1 相文件（功率/导出量）：3, 7, 11, 15, 19, 23, 27, 31

### 4.4 本地接收目录数据

#### 4.4.1 `本地接收目录表示` — 模拟量

- 文件：`DCBSDYAnalog(YYYY_MM_DD_HH).dat`
- 84 个文件，492,672 条采样
- 记录结构（约 16 字节）：
  - +0：4 字节时间戳（小端 uint32）
  - +4：2 字节通道（通常 0）
  - +6：2 字节测量类型（1~32）
  - +12：4 字节 float32 值
- 主要测量类型：1、2、3、4、5、6、32 等
- 值域多在 0~63 V 之间，疑似表示电压

#### 4.4.2 `本地接收目录扳动` — 开关量

- 文件：`Digit(YYYY_MM_DD_HH).dat`
- 84 个文件，13,667 条事件
- 记录结构（变长）：
  - 4 字节时间戳（小端 uint32）
  - 2 字节 record_type / 计数（大端 uint16）
  - N×2 字节点数据（大端 uint16），高字节为 state_byte，低字节为 point_id
  - 1 字节校验/填充
- 原始值：`raw_value = (state_byte << 8) | point_id`
- 稳定状态：`0x2f` 与 `0x00`
- 高频 point_id：168、170、174、175、184、185、186、187

## 5. 现场软件分析

### 5.1 软件定位

`本地接收目录/` 下是一套 **铁路信号集中监测系统（CSM）** 站机软件，用于对车站信号设备（道岔、轨道电路、信号机、电源屏、列控中心等）进行实时监测、报警、数据记录和回放。

### 5.2 主要可执行程序

| 程序 | 作用 |
|---|---|
| `CsmthStation.exe` | 站机主程序，负责数据采集与处理 |
| `CsmthStationMonitor.exe` | 站机监控/显示程序 |
| `CsmthStation_Nbr.exe` | 邻居站/备用程序 |
| `CsmthReplay.exe` / `CsmthReplayMonitor.exe` | 历史数据回放 |
| `AutoUpdate.exe` | 自动更新 |
| `Daemon.exe` | 守护进程 |

### 5.3 架构特点

- **进程模型**：`AppInfo.ini` 定义 `CsmthStationMonitor` 和 `CsmthStation` 两个核心进程
- **通信平台**：`CommPlatform.xml` 统一配置各进程间实时通信服务（`rt`）
- **插件化**：`PlugIn.xml` 为不同程序配置插件 DLL，主要功能包括：
  - 历史数据访问：`HisDataAccessPlug.dll`
  - 实时数据：`RealTimeDataPlug.dll` / `nRealTimeDataPlug.dll`
  - 接口插件：CAN、XY、DYP、CBI/Tk/ADX、Wdw 等
  - 报警、分析、回放、状态图、设备状态图等
- **依赖库**：DevExpress、LiveCharts、Newtonsoft.Json、NLog、ZeroMQ 等
- **界面皮肤**：`Skin/` 目录下大量 XAML/界面资源

### 5.4 车站配置组织

`DataFile/` 目录按车站组织配置：

```
DataFile/
├── Station_DHD
├── Station_DHS
├── Station_MDZ
├── Station_SDB
├── Station_SDY
├── Station_SHZ
├── Station_SJM
├── Station_SM3 ~ SM6
├── Station_SSB          # 三水北
├── Station_SYJ
├── Station_YDH
└── Station_ZQZ
```

每个车站配置文件夹包含：

- `digit.ini`：开关量点表（继电器、道岔表示等）
- `DevVars.xml`：设备变量（道岔、按钮、信号机、轨道电路等）
- `AnalogParaModel_*.INI`：模拟量参数（表示电压、转辙机电压等）
- `DC.ini`：道岔电流/功率通道配置（对应 CSM2010 道岔曲线）
- `CommConf.xml`：该站通信接口配置
- `*StatusGraph.xml`：TCC、CBI、道岔、轨道电路等状态图配置
- `报警配置文件/`：大量报警规则 XML

### 5.5 通信接口

以 `Station_SSB` 为例，接口包括：

- TCC（列控中心）
- CbiTkADX（CBI 接口）
- ZPW2000
- DYP（电源屏）
- CTC
- 单元通信

## 6. 关键发现

### 6.1 解析器与现场软件输出一致

现场软件测试目录下有几份 CSV 输出：

- `本地接收目录扳动_test/digit_test3.csv`
- `本地接收目录表示_test/analog_test.csv`

对比结论：

- `digit_test3.csv` 与我们的 Digit 解析器输出几乎完全一致（91 行 vs 92 行，仅边界差 1 条），说明解析逻辑正确。
- `analog_test.csv` 与我们的模拟量解析结果一致，包括 `channel=555` 的异常样本（可能是二进制边界处的误解析）。
- `digit_test2.csv` 是 signed 16-bit 调试输出，`point_low` 列才是正确的低字节点号。

### 6.2 开关量数据特征

- 高频 point_id：168、170、174、175、184、185、186、187
- 在 `Station_SSB/digit.ini` 中：
  - `point_id 184` → `537GH`
  - `point_id 185` → `537GB`
  - 其余 6 个在配置中名为 `DUMMY`
- 数据呈现明显的“整点脉冲 + 整小时稳定”模式：
  - 每个整点开头有约 1 分钟的快速翻转
  - 之后整小时保持稳定状态
- 170/174/175/184/185/186/187 的稳定状态几乎完全同步（同高同低）
- 168 几乎始终为 `0x00`

### 6.3 与现场核对的矛盾

现场人员提供了 7 个时段的道岔实际位置记录，但将数字量稳定状态与现场位置对比后，**无法建立一一对应关系**：

- 多个不同的现场位置组合对应相同的数字量状态（如“全 0x00”）
- 现场记录的 6-28 09:00~10:00、16:00~17:00 道岔转换过程，在数字量数据中没有被捕捉到（整小时保持稳定）

因此，当前这批 Digit 数据**不能直接作为真实道岔位置记录使用**，更可能是：

- 通信测试/心跳数据
- 配置文件与实际数据来源不一致
- 或 point_id 含义并非预期的 8 组道岔

### 6.4 模拟量数据

- `本地接收目录表示` 中的 type 1 等通道值域在 0~63 V 之间
- 疑似表示电压，但缺少通道与具体道岔的映射
- 模拟量值与数字量状态之间没有稳定、显著的对应关系

## 7. 遗留问题与后续工作

### 7.1 待确认事项

1. **point_id 与道岔的映射**
   - 168、170、174、175、184、185、186、187 是否确实对应 1#J/1#X/2#J/2#X/3#J/3#X/4#J/4#X？
   - 184/185 已对应 537GH/537GB，537 是 1#~4# 中的哪一个？

2. **H/B 含义**
   - 哪个代表定位，哪个代表反位？
   - 需现场确认

3. **数据来源一致性**
   - `本地接收目录扳动` 数据是否为真实道岔动作？
   - 是否与现场核对的时间/来源完全一致？

4. **模拟量通道映射**
   - `本地接收目录表示` 中各 type 对应哪组道岔的表示电压？

### 7.2 后续工作建议

1. **按站点单独配置**
   - 每个车站的 `DataFile/Station_<NAME>/` 都需要单独建立映射
   - 收集每个车站的：
     - point_id ↔ 道岔编号映射表
     - H/B ↔ 定位/反位对应关系
     - 模拟量通道 ↔ 道岔表示电压映射

2. **验证数据真实性**
   - 用 `CsmthReplay.exe` 加载现场数据，与软件界面显示对比
   - 确认 Digit 数据是否真实反映道岔动作

3. **生成最终输出**
   - 一旦映射明确，可生成：
     - 带道岔名称和定/反位标识的 movement 日志
     - 每个道岔的完整动作时间线
     - 数字量与表示电压的联合分析
