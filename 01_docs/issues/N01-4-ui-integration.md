# N01-4: UI 集成 — 站点在线状态 + 切换 + 数据清理

> **状态**: 待实现
> **依赖**: N01-1, N01-2, N01-3
> **阻塞**: 无

## Type

Feature

## What to build

将网络功能集成到 SwitchMonitor.UI，使站点管理、在线监控、数据清理可操作。

### 1. 站点列表（侧边栏改造）

现有侧边栏只显示转辙机列表。新增一级"站点选择"：

```
┌─────────────────┐
│ 站点: [▼ SSB 三水北站 ●] │  ← 下拉切换站点，●绿色=在线
│                 │
│ 转辙机:         │
│   1-J           │
│   1-X           │
│   2-J           │
│   ...           │
└─────────────────┘
```

- 下拉菜单列出 `teamStations`（班组终端）或 `stations`（总终端）中所有站点
- 每个站点条目旁显示在线状态灯（绿=在线 / 红=离线 / 灰=未知）
- 切换站点 → 切换 SQLite 连接 → 刷新转辙机列表和索引
- 复用现有 `SelectedSiteId` + `ResolveParsedDataDir` 机制

### 2. 离线告警

站点离线时：
- 侧边栏状态灯变红
- 右下角弹出气泡提示"XX站 通信中断"（3 秒后自动消失）
- 日志记录离线事件

### 3. 补拉按钮

站点离线恢复后或手动需要时：
- 站点上下文菜单增加"补拉数据"选项
- 或侧边栏离线站点旁显示"补拉"按钮
- 点击后显示进度对话框（非模态）："正在补拉 XX站…已拉 45 条"

### 4. 数据清理对话框

"工具"菜单 → "清理历史数据" → 弹出对话框：

```
清理历史数据
├── 选择站点:
│   ☑ SSB 三水北站 (2.1GB, 最早: 2025-03-15)
│   ☐ DHD 大湖东站 (1.8GB, 最早: 2025-06-01)
│   ☐ MDZ 某站      (0.9GB, 最早: 2025-09-12)
├── 保留最近 [  365  ] 天
├── 预计释放空间: 1.4GB
└── [确认清理] [取消]
```

- 仅总终端角色显示此菜单项
- 清理后刷新站点列表中的数据量显示

### 5. 状态栏增强

现有状态栏增加：
- 当前选中站点名（替换原来的固定文本）
- 最后数据同步时间

## Acceptance criteria

- [ ] 切换站点 → 侧边栏刷新转辙机列表，图表清空等待选择
- [ ] 在线站点显示绿点，离线站点显示红点
- [ ] 离线时弹气泡告警，站机恢复后自动消除
- [ ] 手动补拉触发后显示进度，完成后刷新数据
- [ ] 数据清理对话框正确显示各站点数据量和最早记录
- [ ] 清理执行后过期数据不可查，释放空间符合预期
- [ ] IE8 WebBrowser 环境下站点切换不要触发 JS 错误

## Further notes

- 站点切换复用现有 `OnSwitchSelected` 完整链路，改动仅在一级选择器
- 气泡提示用 `NotifyIcon` + `BalloonTip`，XP 兼容
- 进度对话框用 `BackgroundWorker` + `ProgressBar`，不阻塞 UI
- 数据量获取：`SELECT COUNT(*), MIN(timestamp), MAX(timestamp) FROM events`，大小 = 文件系统文件大小
