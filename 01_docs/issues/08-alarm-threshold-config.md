## What to build

提供独立的报警阈值配置窗口，允许用户修改电流和功率的报警上限值。配置持久化到 `config.json`，修改后图表阈值线实时更新。预留报警下限字段（不做逻辑）。

### 配置窗口

菜单栏 → "报警阈值设置" → 弹出模态对话框：

```
┌─ 报警阈值设置 ────────────────────────┐
│                                       │
│  电流曲线                              │
│  □ 启用报警上限                       │
│  上限值: [2.0  ] A                    │
│  颜色:   [■ 红色]                     │
│  线型:   [▼ 虚线]                     │
│                                       │
│  功率曲线                              │
│  □ 启用报警上限                       │
│  上限值: [1.5  ] KW                   │
│  颜色:   [■ 红色]                     │
│  线型:   [▼ 虚线]                     │
│                                       │
│              [保存]  [取消]            │
└───────────────────────────────────────┘
```

### 配置持久化

- 加载：从 `config.json` 的 `alarmThresholds` 节读取
- 保存：用户点击"保存"→ 更新 `ConfigManager` 内存中的值 → 序列化回 `config.json`
- 取消：关闭窗口，不保存

### 图表实时更新

- 保存后 → C# 检查当前是否有图表已加载
- 如有 → `InvokeScript("updateThreshold", thresholdJson)` → JS 端更新 `yAxis.plotLines`
- 无需重新加载整个图表

### 报警下限

- UI 上放灰显的文本框，标注"预留功能"
- 数据库中预留字段，不做逻辑

## Acceptance criteria

- [ ] 菜单栏可打开"报警阈值设置"对话框
- [ ] 电流/功率阈值各可独立启用/禁用
- [ ] 阈值数值可编辑，范围 0.0 ~ 999.9
- [ ] 保存后 `config.json` 文件更新
- [ ] 保存后图表中红色虚线实时更新到新值
- [ ] 取消不保存修改
- [ ] 程序重启后上次配置的阈值生效
- [ ] 输入非法值（非数字、负数）有提示而不崩溃

## Further notes

- 阈值单位从配置文件中读取（A / KW），修改阈值时不改单位
- 配置文件的 JSON 写回时保持格式和注释
- 对话框用 WinForms `Form` 原生控件（不用 HTML），保证稳定

## 合并说明

本 slice 的**全部功能需求**已合并到 [D5 UI 报警展示 + 诊断参数配置](../diagnosis/issues/D5-ui-alarm-display.md)，
作为其中第 4 节"诊断参数配置对话框"的**图表阈值线子区域**。合并后的设计：

- 诊断规则阈值（R0-R8）→ 走 `thresholds.json → DiagnosisEngine`
- 图表阈值线（电流/功率上限）→ 走 `config.json → JS updateThreshold()`
- **两者在同一窗口中管理**，避免用户困惑"两个阈值设置"

该窗口的图表阈值线部分保留本 slice 的 UI 布局（电流/功率独立启用、上限值编辑、颜色线型），
并在原"保存/取消"基础上增加了"保存并重跑诊断"和"恢复默认"按钮。
