# ZYJ7道岔表示二极管击穿 — 电流曲线特征分析

## 1. 资料概要

本文档来自铁路信号现场维护资料（公众号：铁路信号资料共享交流），以荣家湾站2011年10月26日的ZYJ7道岔故障实例，分析了**表示二极管击穿**在道岔动作电流曲线上的特征表现。

核心发现：表示二极管击穿时，电流曲线**尾部小平台（1DQJ缓放段）电流值翻倍**（~1.7A→~3.4A），而启动峰值和转换段基本正常，动作时间仅略微延长。

## 2. 故障机制分析

### 2.1 电路原理

```
正常状态：表示回路 → 表示二极管正向导通 → 正常电流回路 → 尾部小平台正常
故障状态：表示二极管击穿(短路) → 表示回路阻抗降低 → 尾部段电流增大 → 约翻倍
```

### 2.2 曲线特征对比

| 特征 | 正常曲线 | 故障曲线 | 变化 | 现有规则能否检出 |
|------|---------|---------|------|-----------------|
| 动作时间 | ~7.24s | ~7.40s | +0.16s (+2%) | ❌ R3阈值±0.5s |
| 启动峰值 | ~5.5-6A | ~5.5-6A | 几乎无变化 | ❌ R4不触发 |
| 转换段均值 | 正常 | 正常 | 几乎无变化 | ❌ R5不触发 |
| **尾部小平台** | **~1.7A** | **~3.4A** | **+100% (翻倍)** | ❌ R8阈值仅±30% |
| 全段均值 | 正常 | 略高 | 被稀释 | ❌ 不够灵敏 |

### 2.3 为什么R8漏报

- R8当前阈值: `deviationRatio: 0.3`（±30%偏离）
- 二极管击穿: 尾部小平台升高 **~100%**（2倍于正常值）
- 理论上2.0 > 1.3，但R8是对"缓放段均值"做检查，如果缓放段定义不够精确，且倍增信号被相邻段污染，实际触发率存疑

**更重要的是**：±30%是一个"对称偏离"设计，而二极管击穿的倍增是单向的（只升不降），需要单向倍增检测逻辑。

## 4. 与我们系统的对照分析

| 维度 | 资料 | 我们的系统 | 可迁移性 |
|------|------|-----------|----------|
| 设备类型 | ZYJ7 电液转辙机 | ZYJ7 电液转辙机 | ✅ 完全一致 |
| 信号类型 | 电流曲线 | 功率曲线 | 高 — 功率=电流×电压，倍增模式同步 |
| 故障类型 | 表示二极管击穿(短路) | one-class 异常检测 | 高 — 可作为特定规则 |
| 检测方法 | 时域波形对比+人工经验 | 五阶段物理分割+统计基线 | 高 — 可直接扩展R8 |
| 样本量 | 1个现场案例 | 23,999条正常曲线 | — |
| 关键阈值 | 尾部平台翻倍(~2×基准) | 可设为≥基准×1.8 | 需dryrun确定最优值 |

## 5. 可提取的规则建议

### 建议1（候选规则 CR001 — 尾部小平台倍增检测）

```
规则名称: 表示二极管击穿检测
规则类型: morphology
检测逻辑:
  1. 提取缓放段(尾部小平台)的功率均值: P_tail
  2. 比较参考基准: P_tail_ref
  3. 若 P_tail > P_tail_ref × 1.8 → 预警 "表示二极管击穿可能"
  4. 同时检查: 启动峰值正常(R4不触发) + 转换段正常(R5不触发) → 置信度提升
```

**参数设定依据**：
- `tailMultiplierThreshold: 1.8` — 故障案例中为~2.0×，取1.8留裕度
- `tailMultiplierNote: "加性(O)=不是'多0.5s'这种加性偏移，而是翻倍——属于乘性变化，适用于乘性阈值"`

### 建议2（扩展R8）

将现有R8拆分为：
- **R8a**: 缓放段一般偏离 — `deviationRatio: ±0.3`（维持现有参数）
- **R14**: 缓放段倍增检测 — `tailMultiplierThreshold: 1.8`（新增单向倍增）

### 建议3（联合诊断）

尾部倍增 + 转换段正常 + 启动正常 = 高置信度"表示电路故障"
尾部异常 + 转换段偏高 + 动作延长 = 更可能是机械问题
→ 这是 G5（相邻段联合判定）的思路

## 6. 提取人备注

- **信号类型差异需关注**：资料使用电流曲线（~1.7A/3.4A），我们的系统使用功率曲线。功率=电流×电压，若电压为~380V，则尾部小平台功率基准值完全不同。倍增逻辑（×1.8）在功率域同样成立，但**基准值必须从实际数据中重新计算**，不能直接用资料中的安培值。
- **案例时间久远**：2011年的案例，二极管材料和电路设计可能有变化，但物理原理不变。
- **单案例局限性**：仅1个现场案例，缺少统计分布——这正是M3规则需要dryrun验证的原因。
- **与现有KB001的互补性**：KB001（付雅婷）关注转换段瞬态异常（R9），KB002关注尾部段倍增异常——两个不同的"隐性问题"盲区，两条不同的规则链。
