# HBF 文件格式逆向分析文档

## 概述
HBF 是 CSM 系统中用于存储道岔动作电流/功率曲线的二进制文件格式。
番禺站数据包含 5 个 HBF 文件（每个 ~512MB，预分配空间）。

## 文件结构

### 第一层：文件头
- Offset 0: `hhcsmfzz` (8 bytes magic number)
- 后续包含文件头字段（结构待定）

### 第二层：256 字节目录项
- 位置：文件头之后，约从 offset 0x848 开始
- 每项 256 字节
- 偏移 0: 道岔 ID ASCII 字符串 (如 "1-J", "7-X")
- 偏移 0x70 (112): 52 字节 descriptor，包含 13 个 uint32 字段:
  - F3 (offset +12): 数据大小或偏移
  - **F4 (offset +16): 子索引块的文件偏移** ⭐
  - F5 (offset +20): 索引序号
  - **F6 (offset +24): 数据字节数 = F7 × 32**
  - **F7 (offset +28): 采样点数**
  - F8-F12: 其他字段
- 每个文件含 30 个目录项（对应 30 个道岔）

### 第三层：32 字节子索引记录
位于 F4 指向的偏移。分为三种类型：

#### 类型 A — 标记在开头 (0x1227 位于 offset 0)
```
[4B: 0x00001227 标记] [4B: u1] [4B: u2] [4B: u3] 
[4B: 0x00000000] [4B: 序号] [4B: 常量] [4B: data_ptr]
```
- u2 在块内是常量
- u3 每次递增 2
- 序号每次递增 1
- data_ptr 每次递增 0x1227（指向数据区，多为空）
- 例：0xd9d4（21-J/X 功率）：1005 条记录

#### 类型 B — 标记在结尾 (0x1227 位于 offset 28)
```
[4B: Unix时间戳] [4B: 标记字段] [4B: 递增字段] [4B: 0x00000000]
[4B: 序号] [4B: 常量] [4B: data_ptr] [4B: 0x00001227 标记]
```
- u32[0] 是合法的 Unix 时间戳（2023-2026 范围）
- u32[1] 前缀 0x08DE（结构标记）
- data_ptr 每次递增 0x1227
- 散布在文件各处，非连续

#### 类型 C — 标记在偏移 7 (0x1227 位于 byte 7-10)
```
[2B: 0x0004] [2B: var] [4B: var] [4B: 0x1227在byte7-10]
[4B: var] [4B: 时间戳?] [4B: 小整数] [4B: var] [4B: var]
```
- 32 字节对齐，但 uint32 字段不对齐
- 结构与类型 A/B 不同
- 例：0xc7ad（7-J/X, 9-J/X 功率）：883 条记录

### 标记字节
- 功率文件：`27 12 00 00` (0x1227)
- 电流文件：`77 32 00 00` (0x3277)

### 目录项字段（13 × uint32 @ offset+0x70）
| 字段 | 含义 | 示例 |
|------|------|------|
| F0-F2 | 保留（常为0） | 0 |
| **F3** | 道岔 ID 常量（跨文件不变） | 0x40f = 1-J |
| **F4** | 子索引块文件偏移 | 0x158e5 |
| F5 | 序列号（共享F4的开关递增） | 0 |
| **F6** | 数据总字节数 = F7 × 32 | 142176 |
| **F7** | 总采样数（32B记录数） | 4443 |
| F8 | = F7（备用） | 4443 |
| **F9** | **★ 采样数据文件偏移（float32 LE）** | 0x13b0add |
| F10 | 保留（0） | 0 |
| F11 | 0x100（256） | 256 |
| F12 | 保留（0） | 0 |

### 采样数据格式（F9 指向）
- **float32 little-endian**，32 字节/采样记录
- 每记录 8 个 float32 通道（32B = 8×4B）
- 哨兵值：-71.622（`0xc28f3e75`）表示无效/空数据
- 功率数据通常在通道 4/5（偏移 16-23 字节）
- 配置参考：1032 采样点/事件，40ms 间隔，0-5KW 范围

### 未完全解决 ⚠️
1. **功率 1.hbf**：仅含索引/元数据，F9 数据全为零
2. **功率 2.hbf**：含实际采样数据，但大多数记录为零或哨兵值
   - 仅 8 条有效功率曲线被提取（35-45 点/条）
   - 系统可能运行时间短，事件积累少
3. **时间戳集成**：子索引记录（F4）含时间戳，尚未与采样数据关联
4. **电流文件**：标记不同（0x3277），结构与功率文件类似但尚未解析
